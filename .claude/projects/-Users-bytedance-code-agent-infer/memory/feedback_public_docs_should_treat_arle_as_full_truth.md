# Rule

Current public docs and site copy must treat `ARLE` as the project name and
`arle` as the user-facing front door.

# Why

The user explicitly corrected the documentation rule: current truth should not
frame `agent-infer` as an active compatibility name in public docs. Public
surfaces should read as one unified project with one unified user-facing CLI.

# Preventive action

- Use `ARLE` in titles, summaries, and project-positioning text.
- Use `arle` for user-facing CLI commands and examples.
- Avoid explaining current public truth in terms of legacy or compatibility
  names unless a historical document truly needs that context.
