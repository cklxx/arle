pure-Rust LLM inference engine — KV-cache-first, built for multi-turn agents.

[synopsis] [performance] [backends] [quickstart] [architecture] [github]

NAME

agent-infer — continuous-batching LLM inference engine in pure Rust. CUDA is the primary serving path; Metal runs on Apple Silicon via an MLX C++ bridge. OpenAI-compatible HTTP API, built-in agent runtime.

SYNOPSIS

$ infer --model-path Qwen3-4B --port 8000      # cuda / linux
$ ./scripts/start_metal_serve.sh               # apple silicon
$ curl localhost:8000/v1/chat/completions \
    -H 'content-type: application/json' \
    -d '{"messages":[{"role":"user","content":"hi"}]}'

DESCRIPTION

Every agent turn pays a prefill tax: system prompt + conversation history + tool results get re-processed as context grows. agent-infer treats prefix-KV reuse as the core problem — everything else is consequence.

Multi-turn KV reuse slot-sticky prefix match; only the new user message prefills.
Token-level KV pool page_size=1, SGLang-style. Zero fragmentation, O(1) alloc/free.
GPU→CPU offload oldest blocks migrate to host RAM; prefetched before attention.
Copy-on-write blocks paged blocks with ref-counting; N requests share 1× prefix.
CUDA Graph decode one captured graph per batch size; 504 launches → 1 replay.

QUICKSTART

— CUDA / Linux

# docker (recommended)
$ docker run --gpus all -v $PWD/Qwen3-4B:/model \
    ghcr.io/cklxx/agent-infer:latest --model-path /model --port 8000

# from source
$ git clone https://github.com/cklxx/agent-infer && cd agent-infer
$ cargo build -p infer --release
$ ./target/release/infer --model-path /path/to/Qwen3-4B --port 8000

— Metal / Apple Silicon

# one-command serve (defaults to Qwen3-0.6B-4bit on :8000)
$ ./scripts/start_metal_serve.sh

# speculative decode (default-on, Qwen3.5 target + draft)
$ ./scripts/run_dflash.sh              # serve
$ ./scripts/run_dflash.sh bench        # baseline vs dflash

prerequisites: CUDA 12.x · Rust 1.85+ · Python 3.10+ with flashinfer-python (build-time only). Apple Silicon needs only Rust + Xcode CLT.

API

OpenAI-compatible. Stable surface: POST /v1/completions, POST /v1/chat/completions, GET /v1/models. Beta: POST /v1/responses (non-streaming subset + SSE output_text.delta). Ops: GET /metrics (Prometheus), GET /v1/stats. 

$ curl localhost:8000/v1/chat/completions \
    -d '{"messages":[{"role":"user","content":"explain kv caching"}],"stream":true}'
