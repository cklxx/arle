NAME

ARLE — a runtime-first Rust workspace. infer is the dedicated serving binary; arle is the front door for local agent execution, training, evaluation, and dataset work.

This page stays deliberately stable. Detailed product truth lives in the repo docs linked below.

AT A GLANCE

— Runtime Spine

Serving remains primary. infer is the canonical HTTP surface for generation, metrics, stats, and sessions.

— One Front Door

arle run, arle train, and arle data give users one stable front door instead of a scatter of task-specific binaries.

— Shared Authority

Runtime code, model loading, train/eval flows, and public docs stay in one Rust workspace so operator truth does not drift.

SYNOPSIS

$ cargo build -p infer --release
$ ./target/release/infer --model-path /path/to/Qwen3-4B --port 8000
$ cargo build --release --features cli --bin arle
$ ./target/release/arle --doctor
$ ./target/release/arle --model-path /path/to/Qwen3-4B \
    run --prompt "Summarize this repo"
$ ./target/release/arle train env
$ ./target/release/arle data convert --help

SURFACES

inferOpenAI-compatible HTTP serving, metrics, stats, and session endpoints on the runtime's canonical server surface.
arle runInteractive REPL and one-shot prompt execution with the same Rust runtime authority behind the local agent loop.
arle trainPretrain, SFT, GRPO, multi-turn RL, and eval workflows through one front door instead of scattered train binaries.
arle dataDataset download and schema conversion utilities that stay versioned with the same workspace and docs.

STATUS

Project postureRuntime-first. infer remains the primary serving surface; arle extends that same runtime into agent, train, eval, and data workflows.
BackendsCUDA is the stable primary serving path on Linux + NVIDIA. Metal is beta on Apple Silicon. CPU is development-only for smoke coverage.
ModelsQwen3 and Qwen3.5-4B ship today. Llama 3 / 4 and DeepSeek V3 / R1 remain planned, not implied.
HTTP/v1/completions, /v1/chat/completions, /v1/models, /metrics, and /v1/stats are the stable public serving surface.
Bench programDated snapshots live under docs/experience/wins/; active CUDA closure work lives in the current decode-alignment plan.

QUICKSTART

— Local CLI smoke

$ git clone https://github.com/cklxx/arle
$ cd arle
$ ./setup.sh
$ cargo build --release --no-default-features \
    --features cpu,no-cuda,cli --bin arle
$ ./target/release/arle --doctor
$ ./target/release/arle --model-path /path/to/Qwen3-0.6B \
    run --prompt "Say hello in one sentence"
$ ./target/release/arle train env

— Serving smoke

$ cargo build -p infer --release
$ ./target/release/infer --model-path /path/to/Qwen3-4B --port 8000
$ curl http://127.0.0.1:8000/v1/chat/completions \
    -H 'Content-Type: application/json' \
    -d '{"messages":[{"role":"user","content":"Hello"}],"max_tokens":64}'

Use --features cli for the default CUDA build, metal,no-cuda,cli on Apple Silicon, and cpu,no-cuda,cli for portable smoke paths. ./setup.sh bootstraps Rust, Python, Zig, and local checks.
