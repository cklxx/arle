[概要] [状态] [快速开始] [文档] [github] [发版] en · zh

名称

ARLE — 以 Rust-native inference runtime 为主干，并在其上集成本地 agent、训练与评测工作流。CUDA 是主部署路径；Metal 和 CPU 冒烟路径与 serving 共用同一套 workspace、模型代码和工具链。

这个站点现在只保留稳定入口职责。更细的产品事实，请直接看下面列出的仓库文档。

概要

$ docker run --gpus all -v /path/to/Qwen3-4B:/model \
    ghcr.io/cklxx/agent-infer:latest --model-path /model --port 8000
$ cargo build -p infer --release
$ ./target/release/infer --model-path /path/to/Qwen3-4B --port 8000
$ cargo build -p agent-infer --release --features cli --bin arle
$ ./target/release/arle --model-path /path/to/Qwen3-4B --max-turns 10

状态

CUDALinux + NVIDIA 上的稳定主 serving 路径。
MetalApple Silicon 上的 Beta 路径，带 live runtime、DFlash 与 MLX 桥接。
CPU仅开发态冒烟路径，用于请求面验证，不是部署目标。
Benchmark带日期的快照集中放在 docs/experience/wins/；当前 CUDA 收口工作见 当前 decode 对齐计划。

快速开始

— 源码构建

$ git clone https://github.com/cklxx/arle
$ cd arle
$ ./setup.sh
$ cargo build -p infer --release
$ ./target/release/infer --model-path /path/to/Qwen3-4B --port 8000

— API 冒烟

$ curl http://localhost:8000/v1/chat/completions \
    -H 'Content-Type: application/json' \
    -d '{"messages":[{"role":"user","content":"你好"}],"max_tokens":64}'

请使用 rust-toolchain.toml 固定的 Rust 工具链。 ./setup.sh 会引导 Rust、Python、Zig 和本地检查。
