名称

ARLE — 以 runtime 为主干的 Rust workspace。infer 是专门的 serving 二进制；arle 是本地 agent、训练、评测与 数据工作的统一前门。

这个页面只保留稳定入口职责；更细的产品真相请直接看下面列出的仓库文档。

概览

— Runtime 主干

Serving 仍是主线。infer 是生成、metrics、stats 与 session 的标准 HTTP 服务面。

— 一张前门

arle run、arle train 与 arle data 把用户入口收成一张前门，不再分散成多组二进制。

— 同一套权威

运行时代码、模型加载、train/eval 流程与公共文档都落在同一套 Rust workspace 里，避免口径漂移。

概要

$ cargo build -p infer --release
$ ./target/release/infer --model-path /path/to/Qwen3-4B --port 8000
$ cargo build --release --features cli --bin arle
$ ./target/release/arle --doctor
$ ./target/release/arle --model-path /path/to/Qwen3-4B \
    run --prompt "总结一下这个仓库"
$ ./target/release/arle train env
$ ./target/release/arle data convert --help

入口面

inferOpenAI 兼容 HTTP serving、metrics、stats 与 session 接口都收口在运行时的标准服务面上。
arle run交互式 REPL 与一次性 prompt 执行，共用同一套 Rust 运行时权威，直接面向本地 agent 工作流。
arle trainPretrain、SFT、GRPO、多轮 RL 与 eval 共用一张前门，不再依赖分散的训练二进制。
arle data数据集下载与 schema 转换工具，和同一套 workspace、版本、文档一起演进。

状态

项目姿态Runtime-first。infer 仍是主 serving 面；arle 在同一套运行时之上向 agent、train、eval、data 扩展。
后端CUDA 是 Linux + NVIDIA 上的稳定主 serving 路径。Metal 在 Apple Silicon 上是 beta。CPU 仅用于开发态冒烟覆盖。
模型当前已交付 Qwen3 与 Qwen3.5-4B。Llama 3 / 4 与 DeepSeek V3 / R1 仍是规划中，不做暗示性承诺。
HTTP/v1/completions、/v1/chat/completions、/v1/models、/metrics 与 /v1/stats 是稳定公共服务面。
Benchmark带日期的快照集中放在 docs/experience/wins/；当前 CUDA 收口工作见 当前 decode 对齐计划。

快速开始

— 本地 CLI 冒烟

$ git clone https://github.com/cklxx/arle
$ cd arle
$ ./setup.sh
$ cargo build --release --no-default-features \
    --features cpu,no-cuda,cli --bin arle
$ ./target/release/arle --doctor
$ ./target/release/arle --model-path /path/to/Qwen3-0.6B \
    run --prompt "用一句话打个招呼"
$ ./target/release/arle train env

— Serving 冒烟

$ cargo build -p infer --release
$ ./target/release/infer --model-path /path/to/Qwen3-4B --port 8000
$ curl http://127.0.0.1:8000/v1/chat/completions \
    -H 'Content-Type: application/json' \
    -d '{"messages":[{"role":"user","content":"你好"}],"max_tokens":64}'

默认 CUDA CLI 用 --features cli，Apple Silicon 用 metal,no-cuda,cli，便携冒烟路径用 cpu,no-cuda,cli。./setup.sh 会引导 Rust、Python、Zig 和本地检查。
