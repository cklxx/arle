名称

ARLE — 纯 Rust 的 agent reinforcement learning engine。 CUDA 为主部署路径；Apple Silicon 通过 MLX C++ 桥接运行 Metal。 OpenAI 兼容的 HTTP 接口，内置 agent 运行时，训练 / 评测 / data 工作流也在 同一个 workspace 内。

概要

$ infer --model-path Qwen3-4B --port 8000      # cuda / linux
$ ./target/release/arle --model-path Qwen3-4B --max-turns 10   # 本地 agent runtime
$ ./target/release/arle train env              # train / eval 前门
$ ./scripts/start_metal_serve.sh               # apple silicon
$ curl localhost:8000/v1/chat/completions \
    -H 'content-type: application/json' \
    -d '{"messages":[{"role":"user","content":"你好"}]}'

描述

智能体每一轮都要付 prefill 税：系统提示 + 历史对话 + 工具结果都会被 重新处理。ARLE 把前缀 KV 复用当成 serving 核心，再在这条运行时主干上 往外长出更完整的 agent reinforcement learning engine。infer 是 HTTP 服务面；arle 是本地 agent / train / data 前门。

多轮 KV 复用 slot-sticky 前缀匹配；每轮只对新用户消息做 prefill。
Token 级 KV 池 page_size=1，SGLang 风格。零碎片，O(1) 分配/释放。
GPU→CPU 卸载 最旧的块迁移到主机内存；注意力前预取回来。
Copy-on-Write 块 分页块 + 引用计数；N 个请求共享 1× 前缀。
CUDA Graph 解码 每个 batch size 一张图；504 次 launch → 1 次 replay。

快速开始

— CUDA / Linux

# docker（推荐）
$ docker run --gpus all -v $PWD/Qwen3-4B:/model \
    ghcr.io/cklxx/agent-infer:latest --model-path /model --port 8000

# 从源码
$ git clone https://github.com/cklxx/agent-infer && cd agent-infer
$ cargo build -p infer --release
$ ./target/release/infer --model-path /path/to/Qwen3-4B --port 8000

— Metal / Apple Silicon

# 一条命令起服务（默认 Qwen3-0.6B-4bit，监听 :8000）
$ ./scripts/start_metal_serve.sh

# 推测解码（Qwen3.5 目标 + 草稿，默认开启）
$ ./scripts/run_dflash.sh              # 起服务
$ ./scripts/run_dflash.sh bench        # 基线 vs dflash

前置要求：CUDA 12.x · Rust 1.85+ · Python 3.10+ 带 flashinfer-python（仅构建期）。Apple Silicon 只需 Rust + Xcode CLT。

API

Serving 只是 ARLE 的一个对外面。完整契约见 docs/http-api.md： POST /v1/completions、 POST /v1/chat/completions、 POST /v1/responses、GET /v1/models、 GET /healthz、GET /readyz、 GET /metrics、GET /v1/stats，以及 /v1/sessions/{session_id}/* 下的会话持久化接口。 

$ curl localhost:8000/v1/chat/completions \
    -d '{"messages":[{"role":"user","content":"解释一下 kv 缓存"}],"stream":true}'
