### 1.1 关键架构特征(DSV4 系)

- **混合注意力**(per-layer `compress_ratios`):
  - `0` → SWA(window 64)— 出现在 layer 0/1 + 末层
  - `4` → CSA(stride-4 compressor + Lightning Indexer top-128)— 偶数层(2,4,...,22 除末)
  - `96` → HCA(stride-96 dense compressed)— 奇数层(3,5,...,21)
  - V4 **abandons MLA**;改用 Q-LoRA(384)+ 单 KV 头(GQA num_kv_heads=1)+ O-LoRA grouping(rank 384,o_groups=4)
- **Lightning Indexer**:`index_n_heads=8`,`index_head_dim=64`,`index_topk=128`,
  `num_hash_layers=2` — 选每个 query 的 top-128 compressed positions
- **MoE**:16 routed + 1 shared,top-2/16 路由,`moe_intermediate_size=512`,
  scoring `sqrtsoftplus`(V4 新),topk_method `noaux_tc`(无辅助损失)
- **Manifold-Constrained Hyper-Connections**:`hc_mult=4` 流,Sinkhorn 20 步
  投影到 Birkhoff polytope
- **MTP**:`num_nextn_predict_layers=1`(speculative-decoding-friendly)
- **YaRN**:base 65536 × factor 16 = **1,048,576 tokens** 有效上下文
- **RoPE 双 theta**:dense 路径 `rope_theta=10000`,compressed 路径
  `compress_rope_theta=160000`(stride 不同需分开 base)

### 1.2 Param 估算(BF16)

> ARLE pre-train + serve 的 sanity check 用,**不替代权威 config**。

| Block | 估算 |
|---|---:|
| Embedding(`129280 × 1024`,not tied)| ~133 M |
| Per-layer attention(q_lora 384 + KV 1 head + o_lora 384×4 groups + indexer)| ~6 M × 24 = ~144 M |
| Per-layer MoE(16 routed × `2 × 1024 × 512` + 1 shared × same + router)| ~17 M × 24 = ~410 M |
| MTP 头(1 transformer block + eh_proj)| ~17 M |
| LM head(not tied,`1024 × 129280`)| ~132 M |
| **总参数** | **~836 M** |
| **Activated/token**(top-2 of 16 + shared + dense)| **~280 M** |

**注**:HF 标的"1B"为粗略 round。实测约 836M params ≈ 0.84B。Activated
ratio 280/836 ≈ 33.5%。

## 4. 战略含义

### 4.1 推理侧(coding/agent runtime)与训练侧(DSV4 from-scratch)双线

- 推理侧 P0 = Coding/Agent workload(W3/W4 bench,Prefill CUDA Graph capture)
  — 见 [`2026-05-07-arle-as-coding-agent-runtime-strategic-pivot.md`](./2026-05-07-arle-as-coding-agent-runtime-strategic-pivot.md)
- 训练侧 P0 = **DSV4 架构 from-scratch repro**(本 doc)
  — 主 plan [`docs/plans/dsv4-small-repro.md`](../plans/dsv4-small-repro.md)

### 4.2 交集 — MoE 推理 + 训练的同代码路径

DSV4 训练完之后,**ARLE 推理侧也能服务 DSV4 模型**(serving DSV4-mini-1B
weights 通过 ARLE Rust runtime)— 这是**真正的"训练-推理一体"**护城河:
内部训出的模型直接用 ARLE 服务,不经过 PyTorch/HF transformers 中转。

### 4.3 与 Qwen3 工作的关系

- Qwen3 路径(`crates/qwen3-spec/`,`crates/qwen35-spec/`)— 推理侧 W3/W4
  bench 用 Qwen3-4B,跟 DSV4 工作**互不冲突**
- 长期:DSV4-mini 训完后接入 W3/W4 bench(自训模型 vs SGLang/vLLM 服务的
  开源模型)— **product-market fit 验证**

## 6. 已知不确定性 & 风险

| 不确定性 | 影响 | 解除条件 |
|---|---|---|
| HF replica config 是否 byte-exact 等于 DSV4-Pro/Flash 缩比版 | 我们的"1:1 重现"声明 | 对照 DeepSeek 官方 config + 技术报告 |
| `hc_mult` `scoring_func` 等新字段在 transformers 库的实现细节 | 推理 numerics | transformers ≥ 4.57.1 源码 review |
| FP4 expert(`expert_dtype=fp4`)在 sm_89 RTX 4070Ti SUPER 不支持 | 训练完整度 | 用 FP8 expert 替代(已在 plan §4.2)|
| Muon optimizer Rust 实现复杂度 | 训练栈 LOC | 参考 Moonlight paper + 现有 PyTorch impl |
| Sinkhorn 投影 forward overhead | 训练 step time | 4×4 矩阵,Sinkhorn 20 步,微秒级,可忽略 |
