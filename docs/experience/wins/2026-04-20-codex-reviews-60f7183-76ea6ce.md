## 76ea6ce — `feat(cli): streaming chat mode default + Ctrl-C cancel + multi-line input`

**Verdict: 3 findings — High / Medium / Medium.**

### High — Ctrl-C does not actually cancel an in-flight generation
`crates/cli/src/repl.rs:628` — the delta-drain loop exits on `cancel`
but `repl.rs:657` immediately `.join()`s the worker thread. Since
`engine.complete_stream()` has no cancellation hook, the REPL still
blocks until the full decode finishes. The double-tap exit path at
`repl.rs:418` is also effectively blocked during generation because
`EXIT_REQUESTED` is only checked after the turn returns to the outer
loop. Commit message's "Ctrl-C cancels the in-flight generation
without exiting" is false as shipped.

### Medium — Cancel path buffers remaining completion in memory
`crates/cli/src/repl.rs:585` comment says the worker drains into a
dropped receiver, but `rx` is still alive through the `.join()` at
`repl.rs:612` and `repl.rs:657`. Because it's an unbounded channel,
a long remaining completion queues in memory even though the UI
already reported cancellation.

### Medium — `/chat` and `/agent` mode switches don't preserve context
`crates/cli/src/repl.rs:365` / `repl.rs:598` (chat_history) vs
`repl.rs:364` / `repl.rs:748` (AgentSession). Switch commands at
`repl.rs:894` only flip `mode`; they do not migrate/unify history.
`/load` at `repl.rs:882` only restores the agent side. Follow-up
after switching modes loses prior conversation context.

## Rule

**Ship claims that match shipped behavior.** If a commit body says
"Ctrl-C cancels," either wire the cancellation all the way through
(cancel token on engine.complete_stream + drop rx + abandon the
join), or narrow the claim to what actually works ("Ctrl-C queues a
cancel that takes effect after the current token batch"). Codex
reviewers will catch the gap; users discovering it in production
erodes trust faster than a smaller honest claim.

## Follow-ups

- **76ea6ce High** fix: engine-level cancel hook on
  `InferenceEngine::complete_stream` OR at minimum drop `rx` on
  cancel + do not `.join()` the worker (let it complete in the
  background, free the REPL immediately). Cross-ref: directly on
  path of user's 2026-04-20 CLI DX directive.
- **76ea6ce Medium #1** falls out of the same fix (drop rx → tx
  fails → worker exits early).
- **76ea6ce Medium #2** (mode-switch context loss) is separate
  design work; park until user decides whether chat and agent
  *should* share history (not obvious).
