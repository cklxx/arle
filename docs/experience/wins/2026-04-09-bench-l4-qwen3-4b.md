# 2026-04-09 · Qwen3-4B L4-24GB Benchmark (post-optimization session)

## Context
Benchmark after optimization session: prefix cache fix, TurboQuant Phase 2, dual-stream architecture, adaptive prefill rate, scheduler preemption.

## Environment
- GPU: NVIDIA L4 24GB
- Model: Qwen3-4B BF16 (7.6GB weights)
- Slots: 7 (auto-computed)
- Max seq len: 4096
- Pool: 12600 tokens, 1.9GB BF16
- CUDA Graph: batch sizes 1, 2, 4

## What Worked
- ITL extremely consistent (~33-36ms) across all input lengths and concurrency
- Near-linear throughput scaling: C=1→C=2 = 1.9x, C=2→C=4 = 2.0x
- TTFT scales linearly with input length as expected
- Prefix cache working correctly for repeated prompts

## Known Issue
- C=8+ triggers CUDA_ERROR_ILLEGAL_ADDRESS (pre-existing, likely OOM at 7 slots with 8+ concurrent requests)

## Rule
- L4 24GB is memory-constrained for Qwen3-4B BF16 (only 7 slots). INT8/W4 quantization would enable more slots.
- ITL bottleneck is the decode GEMV (~33ms on L4), not scheduler overhead.
